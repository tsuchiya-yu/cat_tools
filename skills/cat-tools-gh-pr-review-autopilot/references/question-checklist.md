# 実装前の確認質問チェックリスト（PRレビュー対応）

次の項目に不明点がある場合は、実装前に質問する。

## 必須項目

- 目的: どの未解決スレッドをどういう状態にすれば完了か
- 対象範囲: どの指摘を修正対象とし、どこまで除外するか
- 受け入れ条件: 完了判定の条件は何か
- 返信ルール: 1行目 `@<reviewer>` 必須、2行目以降の期待内容
- ブランチ運用: 既存ブランチ push を前提にしてよいか
- 判定停止条件: 迷うケースでユーザー相談して停止する条件が明確か
- 検証方法: 未指定時は `npm run lint` + `npm run test` を実行し、画面変更時は Playwright MCP で対象ページキャプチャ取得
- 制約: 未指定時は「レビュー指摘対応に必要な最小変更のみ」を適用

## 任意項目

- 優先度の方針: 速度優先か、堅牢性優先か
- 後方互換性の要件
- リリース時期や期限の制約

## 判定ルール

- 目的 / 対象範囲 / 受け入れ条件 / 返信ルール / ブランチ運用 / 判定停止条件 のいずれかが未確定なら、質問して実装を停止する。
- 検証方法 / 制約 はデフォルト方針を適用できる場合、未確定扱いにしない。
- PR本文やレビューコメントの明示要件がデフォルト方針と衝突する場合は、明示要件を優先する。
