# 実装前の確認質問チェックリスト

次の項目に不明点がある場合は、実装前に質問する。

## 必須項目

- 目的: 何を達成すれば完了か
- 対象範囲: どの機能/ファイルが対象で、どこまで除外か
- 受け入れ条件: 完了判定の条件は何か
- 検証方法: 未指定時は `npm run lint` + `npm run test` を実行し、画面変更時は Playwright MCP で対象ページキャプチャ取得
- 制約: 未指定時は「Issue対応に必要な最小変更のみ」を適用

## 任意項目

- 優先度の方針: 速度優先か、堅牢性優先か
- 後方互換性の要件
- リリース時期や期限の制約

## 判定ルール

- 目的 / 対象範囲 / 受け入れ条件 のいずれかが未確定なら、質問して実装を停止する。
- 検証方法 / 制約 はデフォルト方針を適用できる場合、未確定扱いにしない。
- Issue本文に明示された要件がデフォルト方針と衝突する場合は、Issue本文を優先する。
